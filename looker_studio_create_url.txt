# 이 파일은 생성물입니다 (git에 커밋된 값은 자리표시자).
#
# Looker Studio는 차트가 배치된 리포트를 코드로 맨바닥에서 만드는 API가 없습니다.
# (커스텀 별칭을 from-scratch로 넣으면 "is not a valid data source alias" 오류.)
# 따라서 완성형 대시보드는 "템플릿 리포트 복제" 방식으로만 자동 생성됩니다.
#
# 사용법:
#   1) looker_studio_setup.md 섹션 0을 따라 대시보드를 1회 수동 제작하고,
#      각 데이터 소스 별칭(Alias)을 뷰 이름(v_daily_queries 등)으로 맞춘 뒤,
#      리포트 URL에서 report id를 복사합니다. (.../reporting/<REPORT_ID>/page/...)
#   2) 템플릿 id를 지정해 배포하면 deploy.sh 가 이 파일에 복제 URL을 덮어씁니다:
#        terraform -chdir=terraform apply \
#          -var="project_id=<PROJECT_ID>" \
#          -var="looker_studio_template_report_id=<REPORT_ID>"
#   3) 생성된 URL을 한 줄 그대로 브라우저에 붙여넣으면 완성형 대시보드가 복제됩니다.
#
# 템플릿 id 없이 배포하면 deploy.sh 는 이 파일 대신 수동 생성 안내를 출력합니다.
